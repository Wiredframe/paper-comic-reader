# App Store submission kit

Everything for shipping Paper Comic Reader 1.0.0 that can be prepared **before** the Apple
Developer account is active. Once the account is live, this folder is your paste-ready
source for App Store Connect (ASC).

```
AppStore/
  metadata.md          Listing text: name, subtitle, keywords, description (EN+DE),
                       what's-new, category, age rating, IAP details, URLs.
  AppReviewNotes.md    App Review notes + why the demo file matters.
  demo/Inklings.cbz    Original sample comic (1.3 MB, JPEG). Attach in ASC → App
                       Review Information, and use it to test the app yourself.
  screenshots/
    iphone-6.9/        6.9" iPhone shots (1320×2868) — required.
    ipad-13/           13" iPad shots (2064×2752) — required for a universal app.
  tools/               Reproducible generators (see below).
```

## The sample comic ("INKLINGS #1")

100% original artwork, generated by `tools/gencomic.swift` (abstract geometric
characters, brand-matched yellow + speech bubbles). Because it's ours, it is
rights-clean for both App Store screenshots and the reviewer demo — no third-party
comic art appears anywhere. `demo/Inklings.cbz` is a lean JPEG build for attaching;
`tools/gencomic.swift` regenerates crisp 4000×6000 PNG pages if needed.

## How the screenshots are made

Real app screenshots, captured from the Simulator — no mockups. The app has tiny
**DEBUG-only** hooks (`ComicReader/Debug/ScreenshotSupport.swift`, compiled out of
Release) that read launch-environment variables so the capture script can place the
app in a known state without tapping through the UI:

| Env var (`SIMCTL_CHILD_…`) | Effect |
|---|---|
| `SEED_LIBRARY_DIR` | Import every .cbz in a folder (fills the Library grid). |
| `SEED_COMIC_PATH`  | Import one comic (for the reader shots). |
| `SCREENSHOT_OPEN_PAGE` | Auto-open the first comic at this page. |
| `SCREENSHOT_TAB`   | Initial tab (`library` / `settings` / …). |
| `SCREENSHOT_TIPS`  | Present the Tip Jar sheet on launch (IAP review shot). |

In a normal Debug run with none of these set, the hooks are inert; in Release they
don't exist. `tools/capture_all.sh` builds the state matrix, sets a clean 9:41 status
bar, and grabs both device sizes. (Edit the paths at the top of the script for your
machine; it references a demo library folder + the full-res `Inklings.cbz`.)

Regenerate:

```bash
# 1. sample comic pages + CBZ
swift tools/gencomic.swift /tmp/pages && (cd /tmp/pages && zip ../Inklings.cbz page-*.png)
# 2. extra library covers (optional, for a fuller grid)
swift tools/gencovers.swift /tmp/covers
# 3. build the Debug app, then capture
xcodebuild -scheme ComicReader -configuration Debug -destination 'generic/platform=iOS Simulator' -derivedDataPath build build
bash tools/capture_all.sh
```

## Screenshot set (current)

1. **Library** — populated cover grid.
2. **Reader — splash** (the "paperverse" page).
3. **Reader — panels** (four-panel page).
4. **Reader — dialogue** (great for the Live Text feature).
5. **Settings** — reader/paper toggles + library info.

Plus, in `screenshots/iap-review/`, the **Tip Jar** sheet — the review screenshot ASC
requires for the in-app purchase (captured via `tools/capture_tip.sh`).

Not yet automated (need one more step, or capture by hand in Xcode):
- **Landscape double-page spread** (marquee feature; headless rotation isn't wired).
- Optional marketing polish: device frames + caption headlines over each shot.
